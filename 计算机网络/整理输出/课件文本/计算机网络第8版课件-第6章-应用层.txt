6.1.3  域名服务器
l 实现域名系统使用分布在各地的域名服务器（DNS 服务器）。
l 一个服务器所负责管辖的（或有权限的）范围叫做区 (zone)。
l 各单位根据具体情况来划分自己管辖范围的区。但在一个区中的所
有节点必须是能够连通的。
l 每一个区设置相应的权限域名服务器，用来保存该区中的所有主机
的域名到 IP 地址的映射。
DNS 服务器的管辖范围不是以“域”为单位，
而是以“区”为单位。

1. 根域名服务器
l 最高层次，最为重要。
l 所有根域名服务器都知道所有的顶级域名服务器的域名和 IP 地址。
l 不管是哪一个本地域名服务器，若要对互联网上任何一个域名进行解
析，只要自己无法解析，就首先求助于根域名服务器。
l 若所有的根域名服务器都瘫痪了，整个互联网中的 DNS 系统就无法
工作了。

2. 顶级域名服务器
l 顶级域名服务器（即 TLD 服务器）负责管理在该顶级域名服务器注
册的所有二级域名。
l 当收到 DNS 查询请求时，就给出相应的回答（可能是最后的结果，
也可能是下一步应当找的域名服务器的 IP 地址）。 

--- PAGE 23 ---
谢希仁 编著计算机网络 (第 8 版)
3. 权限域名服务器
l 负责一个区（zone）的域名服务器。
l 当一个权限域名服务器还不能给出最后的查询回答时，就会告诉发
出查询请求的 DNS 客户，下一步应当找哪一个权限域名服务器。 

--- PAGE 24 ---
谢希仁 编著计算机网络 (第 8 版)
4. 本地域名服务器
l 非常重要。
l 当一个主机发出 DNS 查询请求时，该查询请求报文就发送给本地域
名服务器。
l 每一个互联网服务提供者 ISP 或一个大学，都可以拥有一个本地域
名服务器。
l 当所要查询的主机也属于同一个本地 ISP 时，该本地域名服务器立
即就能将所查询的主机名转换为它的 IP 地址，而不需要再去询问其
他的域名服务器。
l 本地域名服务器有时也称为默认域名服务器。

提高域名服务器的可靠性
l DNS 域名服务器都把数据复制到几个域名服务器来保存，其中的一
个是主域名服务器，其他的是辅助域名服务器。
l 当主域名服务器出故障时，辅助域名服务器可以保证 DNS 的查询工
作不会中断。
l 主域名服务器定期把数据复制到辅助域名服务器中，而更改数据只
能在主域名服务器中进行，保证了数据的一致性。

高速缓存
l 也称为高速缓存域名服务器。
l 存放最近用过的名字以及从何处获得名字映射信息的记录。
l 作用：大大减轻根域名服务器的负荷，使 DNS 查询请求和回答报文
的数量大为减少。 
l 域名服务器应为每项内容设置计时器，并处理超过合理时间的项。
l 当权限域名服务器回答一个查询请求时，在响应中指明绑定有效存
在的时间值。增加此时间值可减少网络开销，而减少此时间值可提
高域名转换的准确性。

6.2.1  FTP 概述
l 文件传送协议 FTP (File Transfer Protocol) 是互联网上使用得最
广泛的文件传送协议。
l 提供交互式的访问，允许客户指明文件的类型与格式，并允许文件
具有存取权限。
l 屏蔽了各计算机系统的细节，因而适合于在异构网络中任意计算机
之间传送文件。

l TFTP (Trivial File Transfer Protocol) 是一个很小且易于实现的文

l 优点：（1）可用于 UDP 环境；（2）代码所占的内存较小。

2. 动态万维网文档
l 静态文档：该文档创作完毕后就存放在万维网服务器中，在被用户
浏览的过程中，内容不会改变。 
l 动态文档：文档的内容是在浏览器访问万维网服务器时才由应用程
序动态创建。
l 动态文档和静态文档之间的主要差别体现在服务器端：文档内容的
生成方法不同。从浏览器的角度看，这两种文档并没有区别。

6.5.6  通用互联网邮件扩充 MIME
SMTP 缺点：
l 限于传送 7 位的 ASCII 码，无法传送非 ASCII 编码的信息。
l 不能传送可执行文件或其他的二进制对象。
l 服务器会拒绝超过一定长度的邮件。
l 某些 SMTP 的实现并没有完全按照 [RFC 821] 的 SMTP 标准。
